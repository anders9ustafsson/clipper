Clipper.ZFillFunction

This property is only exposed if the Preprocessor directive use_xyz has been defined. (If it is defined, a Z member will be added to the IntPoint structure.) Once a custom callback function is assigned it can then assign Z values to points of intersection during clipping operations.

During clipping operations vertices added to the solution mostly correspond to matching vertices in the supplied subject and clip polygons. For these vertices the Z values are simply copied from those matching. However at points of intersection where new vertices have to be created, there are no matching vertices in the input polygons and there's no obvious way to assign Z values. It really depends on the needs of the user.

To help the user to provide their own Z values the callback function's first 2 parameters contain the Z values of the vertices on the two paths leading into the intersection. The third parameter ia a reference to the new intersection point, though the user should only modify the Z-member.

 

In the example in the image above, two intersection points (A & B) are created during a clipping operation on two squares. The callback function provides two Z values corresponding to the Z values of the vertices that lead into the intersection. In the case of intersection A, the Z values will be 101 and 2. In the case of intersection B, the Z values will be 104 and 3. It's entirely up to the user to decide what to do with these values.
