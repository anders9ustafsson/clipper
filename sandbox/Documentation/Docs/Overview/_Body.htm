Overview

The Clipper Library performs line and polygon clipping, and line and polygon offsetting.

Several features set Clipper apart from other clipping libraries:

  • it accepts all types of polygons including self-intersecting ones
  • it supports multiple polygon filling rules (EvenOdd, NonZero, Positive, Negative)
  • it's very fast relative to other libraries
  • it also performs line and polygon offsetting
  • it's numerically robust
  • it's free to use in both freeware and commercial applications

Terminology:

  • Clipping: commonly refers to the process of removing from a set 2-dimensional geometric shapes (ie lines and polygons) those parts that are outside a rectangular 'clipping' window. This can be achieved by intersecting subject polygons (and lines) with the clipping rectangle. In a more general sense, the clipping window need not be rectangular but can be any type of polygon, even multiple polygons. Also, while clipping typically refers to an intersection operation, in this documentation it will refer to any one of the four boolean operations (intersection, union, difference and exclusive-or).
  • Path: an ordered sequence of vertices defining a single geometric contour that is either a line (an open path) or a polygon (a closed path).
  • Contour: synonymous with path.
  • Line: an open path representing line segments bounded by 2 or more vertices.
  • Polygon: in a general sense is a two-dimensional region that's bounded by an outer closed path and containing zero to many 'holes'. More specifically in this documentation polygon refers to a single Path that's known to be closed.
  • Hole: a closed region within a polygon that's not part of the polygon.
  • Polygon Filling Rule: determines those regions bounded by contours that are inside polygons and those which are outside. Polygons can be defined by using a relatively complex data structure that includes an outer contour and a list of hole contours. Alternatively, a single list of contours can be used to define any number of polygons as long as a filling rule has been specified. The Clipper Library uses this latter method to define polygons because it's simpler, and because the former method doesn't generally accommodate self-intersecting polygons.

Distribution package contents:

The ZIP package contains the Clipper library's source code, a Windows CHM help file, HTML help, and a number of compiled demo applications (with full source code). The library was initially written in Delphi Pascal (and compiles with Delphi version 7 or above) but now contains C++, C# and Python translations too. The library's source code in each language is about 5000 lines. The Delphi code contains reasonably extensive comments, but comments are fewer in the C++ and C# code. The included sample applications show how Clipper can be used with the different languages using a number of graphics display libraries including - AGG, Cairo, OpenGL, Graphics32, GDI+ and SVG.

References:

The Library is based on but significantly extends Bala Vatti's polygon clipping algorithm as described in "A generic solution to polygon clipping", Communications of the ACM, Vol 35, Issue 7 (July 1992) pp 56-63.

A section in "Computer graphics and geometric modeling: implementation and algorithms" by By Max K. Agoston (Springer, 2005) discussing Vatti Polygon Clipping was also helpful in creating the initial Clipper implementation.

The paper titled "Polygon Offsetting by Computing Winding Numbers" by Chen & McMains (Paper no. DETC2005-85513, ASME 2005. Pages 565-575) contains helpful discussion on the complexities of polygon offsetting together with some solutions.
